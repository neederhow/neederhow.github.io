新建文章与页面

生成文章或页面的模板放在博客文件夹根目录下的 scaffolds 文件夹里面，文章对应的是 post.md ，页面对应的是page.md，草稿的是draft.md

编写文章

“Hexo 使用 Markdown 解析文章”,语法相关请搜索互联网。

写完后按照启动本地服务所写的方法看看效果，使用Markdown编辑器可以在编写时实时看到渲染结果，启动本地服务器的效果经过主题渲染但其实差不多。

插入图片

方法一

在博客根目录的 source 文件夹下新建一个 img 文件夹专门存放图片，在博文中引用的图片路径为 /img/图片名.后缀

部署时会将图片上传到Github，其主机在境外，如果图片过多过大速度很慢，而且项目空间是有限的。而且由于路径是部署后的相对路径，编辑时看不到图片。

方法二

对于那些想要更有规律地提供图片和其他资源以及想要将他们的资源分布在各个文章上的人来说，Hexo也提供了更组织化的方式来管理资源，将站点配置文件中的 post_asset_folder 选项设为 true 来打开文章资源文件夹

然后再博文中通过相对路径引用

没用过这种方式，但文件任然在本地，缺点同方法一

图床

使用七牛云储存，用户实名审核之后有免费的10G空间，新建空间，专门用来放置博客上引用的资源，进入空间后点击「内容管理」，再点击「上传文件」。

而后在下方文件列表中需要的图片右边点击...复制外链。
在文章中引用

解决了方法一，方法二的缺点，编辑时可以看到图片添加后的效果。
使用hexo-admin-qiniu插件可以在线编辑并将图片自动上传七牛云

打开 http://localhost:4000/admin/ 即可
具体看插件主页hexo-admin-qiniu
然而我使用时没有效果，只黏贴了一条文件路径，应该是系统的问题，总之很迷。

安装主题

更多主题请看知乎专栏\: 有哪些好看的 Hexo 主题？
我永远只爱NexT

复制主题

通过Git将主题从Github上克隆到Hexo目录的themes文件夹下

启用主题

编辑站点配置文件_config.yml,找到 theme 字段，并将其值更改为 next

启动本地服务看看效果

更改主题外观

NexT有四个外观，博主用的是 Mist，直接更改主题配置文件themes/next/_config.yml的 scheme 参数即可，如果显示的是繁体中文，那么站点配置文件中的 language: zh-CN

关于NexT的更多设置请参见NexT官方文档。

注册给github并创建pages仓库

在本地运行没有问题后就可以部署到公共网络了，在此之前，先得有服务器让你的项目可以托管，那么Github Page就是个很好的东西，它们可以让我们访问静态文件，而Hexo生成的恰恰是静态文件。具体请查看 GithubPages

注册Github账号

登录Github官网注册，本站与全球各大搜索引擎平台开展了战略合作，不懂的地方请直接使用各大搜索平台。

创建Repository(Github Pages)

登录后在主页可以看到new Repositories，点击创建项目。

github pages其实就是一个特殊的仓库，创建时只需填写Repository name为{user_name}.github.io，其中{user_name}必须与你的用户名一样，这是github pages的特殊命名规范，我已经有了一个pages项目，所以出现红色警告”仓库neederhow.github.io已经存在”

配置SSH与Git

有了服务器后就是将我们的个人电脑与远程服务器取得联系，通过SSH建立安全连接，Git绑定远程仓库。

生成SSH Key

后面的 your_email@youremail.com 改为你的邮箱，之后会要求确认路径和输入密码，我们这使用默认的一路回车就行。成功的话会在~/下生成 .ssh 文件夹，进去，打开 id_rsa.pub，复制里面的key

添加SSH Key

首先是Github，登录Github，右上角 头像 -> Settings —> SSH nd GPG keys —> New SSH key 。把公钥粘贴到key中，填好title并点击 Add SSH key

验证

如果是第一次的会提示是否continue，输入yes就会看到：
